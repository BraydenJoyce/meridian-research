# ADR-002: Database Schema — Research Sessions, Sources, Agent Events

## Status
Accepted

## Context
All durable application state is stored in Supabase PostgreSQL 15. Three tables are needed for Milestone 1:

- `research_sessions` — one row per user-submitted research question; tracks lifecycle from receipt to delivery.
- `sources` — one row per discovered web source associated with a session; stores raw and processed content.
- `agent_events` — append-only event log; one row per meaningful action taken by an agent; drives the SSE trace stream and provides a full audit trail.

Future milestones will add `users`, `subscriptions`, and billing-related tables. The Milestone 1 schema must not require modification to accommodate those additions — they will be added via Alembic migrations.

Constraints:
- All primary keys are UUIDs (v4) generated by the application layer, not PostgreSQL `SERIAL`. This is required for compatibility with Supabase's row-level security patterns in later milestones.
- All timestamps are stored as `TIMESTAMPTZ` (UTC). The application never stores naive datetimes.
- `user_id` columns are `UUID NULLABLE` in Milestone 1. They become non-null and foreign-keyed to `auth.users` in the auth milestone.
- The schema is managed exclusively via Alembic migrations. No manual DDL edits to the live database.

## Decision

The schema below is the authoritative definition. Every column, constraint, and index listed here is created in the Alembic initial migration. No columns are omitted "for later."

```sql
-- ============================================================
-- research_sessions
-- One row per research question submitted by a (future) user.
-- Tracks full lifecycle: queued → running → completed | failed.
-- ============================================================
CREATE TABLE research_sessions (
    id                  UUID            PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id             UUID            NULL,               -- NULL in Milestone 1; FK to auth.users added in auth milestone
    question            TEXT            NOT NULL CHECK (char_length(question) BETWEEN 10 AND 2000),
    status              TEXT            NOT NULL DEFAULT 'queued'
                                            CHECK (status IN ('queued', 'running', 'completed', 'failed')),
    report_markdown     TEXT            NULL,               -- populated by Writer agent on completion
    error_message       TEXT            NULL,               -- populated on failure
    sub_tasks           JSONB           NULL,               -- Planner output: array of sub-task strings
    created_at          TIMESTAMPTZ     NOT NULL DEFAULT NOW(),
    updated_at          TIMESTAMPTZ     NOT NULL DEFAULT NOW(),
    completed_at        TIMESTAMPTZ     NULL
);

-- Fast lookup by status for the worker queue polling fallback
CREATE INDEX idx_research_sessions_status
    ON research_sessions (status)
    WHERE status IN ('queued', 'running');

-- Fast lookup by user for future dashboard queries
CREATE INDEX idx_research_sessions_user_id
    ON research_sessions (user_id)
    WHERE user_id IS NOT NULL;

-- Automatically maintain updated_at on every row update
CREATE OR REPLACE FUNCTION set_updated_at()
RETURNS TRIGGER LANGUAGE plpgsql AS $$
BEGIN
    NEW.updated_at = NOW();
    RETURN NEW;
END;
$$;

CREATE TRIGGER trg_research_sessions_updated_at
    BEFORE UPDATE ON research_sessions
    FOR EACH ROW EXECUTE FUNCTION set_updated_at();


-- ============================================================
-- sources
-- One row per web source discovered during a research session.
-- A session typically accumulates 20–200 sources across all sub-tasks.
-- ============================================================
CREATE TABLE sources (
    id                  UUID            PRIMARY KEY DEFAULT gen_random_uuid(),
    session_id          UUID            NOT NULL REFERENCES research_sessions (id) ON DELETE CASCADE,
    url                 TEXT            NOT NULL,
    title               TEXT            NULL,
    domain              TEXT            NULL,               -- extracted from url (e.g. 'reuters.com')
    sub_task_index      INTEGER         NOT NULL,           -- which Planner sub-task produced this source (0-based)
    raw_content         TEXT            NULL,               -- full scraped text before processing
    cleaned_content     TEXT            NULL,               -- ETL output: deduplicated, normalised text
    relevance_score     NUMERIC(5, 4)   NULL                -- ETL output: 0.0000 to 1.0000
                                            CHECK (relevance_score IS NULL OR relevance_score BETWEEN 0 AND 1),
    entities            JSONB           NULL,               -- ETL output: extracted named entities [{type, value}]
    qdrant_point_id     UUID            NULL,               -- ID of the corresponding vector in Qdrant
    fetched_at          TIMESTAMPTZ     NULL,               -- when raw_content was retrieved
    created_at          TIMESTAMPTZ     NOT NULL DEFAULT NOW(),
    updated_at          TIMESTAMPTZ     NOT NULL DEFAULT NOW()
);

-- Primary access pattern: all sources for a session
CREATE INDEX idx_sources_session_id
    ON sources (session_id);

-- Secondary: sources for a specific sub-task within a session
CREATE INDEX idx_sources_session_sub_task
    ON sources (session_id, sub_task_index);

-- Deduplication: unique URL per session (same URL from different sub-tasks is one row)
CREATE UNIQUE INDEX idx_sources_session_url
    ON sources (session_id, url);

CREATE TRIGGER trg_sources_updated_at
    BEFORE UPDATE ON sources
    FOR EACH ROW EXECUTE FUNCTION set_updated_at();


-- ============================================================
-- agent_events
-- Append-only event log. One row per agent action.
-- Powers the SSE trace stream and provides full audit trail.
-- Never update or delete rows in this table.
-- ============================================================
CREATE TABLE agent_events (
    id                  UUID            PRIMARY KEY DEFAULT gen_random_uuid(),
    session_id          UUID            NOT NULL REFERENCES research_sessions (id) ON DELETE CASCADE,
    agent_type          TEXT            NOT NULL
                                            CHECK (agent_type IN ('planner', 'web_search', 'etl', 'writer', 'system')),
    event_type          TEXT            NOT NULL
                                            CHECK (event_type IN (
                                                'agent_started',
                                                'agent_completed',
                                                'agent_failed',
                                                'sub_task_started',
                                                'sub_task_completed',
                                                'source_fetched',
                                                'etl_progress',
                                                'report_chunk',
                                                'done',
                                                'error'
                                            )),
    payload             JSONB           NOT NULL DEFAULT '{}',  -- event-specific data (see ADR-003)
    sequence_number     INTEGER         NOT NULL,               -- monotonically increasing per session; set by application
    created_at          TIMESTAMPTZ     NOT NULL DEFAULT NOW()
);

-- Primary access pattern: all events for a session in order
CREATE INDEX idx_agent_events_session_seq
    ON agent_events (session_id, sequence_number ASC);

-- Efficient SSE reconnect: events after a given sequence number
CREATE INDEX idx_agent_events_session_seq_filter
    ON agent_events (session_id, sequence_number)
    WHERE sequence_number > 0;
```

### Column semantics and invariants

**research_sessions:**
- `status` transitions are strictly one-way: `queued → running → completed` or `queued → running → failed`. No backward transitions.
- `report_markdown` is NULL until status is `completed`. It is set atomically with the status update.
- `sub_tasks` is the exact JSON output of the Planner agent: `["sub-task string 1", "sub-task string 2", ...]`.
- `error_message` is a human-readable string, not a stack trace. Stack traces go in `agent_events`.

**sources:**
- `url` is stored exactly as returned by the search API (no normalization). Deduplication is by exact URL match within a session.
- `relevance_score` is computed by the ETL agent using BM25 or a lightweight embedding similarity against the original question. It is not the search engine's ranking.
- `qdrant_point_id` is set after the ETL agent upserts the source embedding into Qdrant. It is NULL until that step completes.
- `raw_content` and `cleaned_content` may be NULL if the source URL was unreachable.

**agent_events:**
- `sequence_number` starts at 1 for the first event of a session and increments by 1 for each subsequent event. It is assigned by the application before INSERT, not by PostgreSQL, to guarantee ordering in the SSE stream.
- `payload` structure is defined per `event_type` in ADR-003. Agents must not add arbitrary keys to payload without a corresponding ADR update.
- Rows are never updated or deleted. The table is append-only by convention. Future RLS policies will enforce this.

### Alembic migration

The initial migration file is `backend/alembic/versions/0001_initial_schema.py`. It must create all three tables, all indexes, the `set_updated_at` function, and both triggers. The migration is reversible: the `downgrade()` function drops all objects in reverse dependency order.

## Implementation notes

**SQLAlchemy ORM models location:** `backend/app/models/research.py`

Each table maps to one SQLAlchemy `DeclarativeBase` subclass. Use `mapped_column` and `Mapped` type annotations (SQLAlchemy 2.0 style). Example pattern:

```python
from sqlalchemy.orm import DeclarativeBase, Mapped, mapped_column
from sqlalchemy import Text, String, DateTime, func
import uuid

class Base(DeclarativeBase):
    pass

class ResearchSession(Base):
    __tablename__ = "research_sessions"
    id: Mapped[uuid.UUID] = mapped_column(primary_key=True, default=uuid.uuid4)
    question: Mapped[str] = mapped_column(Text, nullable=False)
    status: Mapped[str] = mapped_column(String(20), nullable=False, default="queued")
    # ... remaining columns
```

**JSONB columns:** Use `sqlalchemy.dialects.postgresql.JSONB` for `sub_tasks`, `entities`, and `payload`.

**Async session factory:** Defined in `backend/app/services/db.py`:
```python
from sqlalchemy.ext.asyncio import create_async_engine, async_sessionmaker
engine = create_async_engine(settings.DATABASE_URL, pool_size=10, max_overflow=20)
AsyncSession = async_sessionmaker(engine, expire_on_commit=False)
```

**sequence_number generation:** The application maintains a per-session counter in Redis (`meridian:seq:{session_uuid}`) using `INCR`. This is atomic and avoids a round-trip to PostgreSQL per event.

## Consequences

**Positive:**
- JSONB for `sub_tasks`, `entities`, and `payload` avoids premature schema normalization while keeping data queryable.
- The `agent_events` append-only log is the single source of truth for SSE replay — no separate event store needed.
- `UNIQUE INDEX` on `(session_id, url)` in `sources` enforces deduplication at the database level, preventing duplicate sources even under concurrent inserts.
- Partial indexes on `research_sessions.status` keep queue polling fast as the table grows.

**Negative / trade-offs:**
- `raw_content` and `cleaned_content` stored as PostgreSQL `TEXT` will cause row bloat for high-volume usage. A future migration should move these columns to object storage (S3/R2) and replace them with URL references. This is a Milestone 3 concern.
- `sequence_number` relies on Redis for generation. If Redis is unavailable mid-session, sequence numbering fails. The worker must abort the session and mark it `failed` in this case.

**Risks:**
- `agent_events` table will grow unboundedly. A retention policy (delete sessions older than 90 days, cascading to events) must be implemented before production launch.
- JSONB payload validation is application-side only. A malformed payload from a buggy agent will be stored without error. ADR-003 defines payload schemas to mitigate this.
